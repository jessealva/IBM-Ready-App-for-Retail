Class: productDetailsCtrl

ReadyAppSummit.milCtrl. productDetailsCtrl

new productDetailsCtrl()

Controller for the product details view that allows you to specific details about a product in Summit, including: reviews, price, similar items, and the ability to add the currently viewed item to a list or to your cart.
Properties:
Name Type Description
addToCart string The text for the 'Add to Cart' button.
addToList string The text for the 'Add to List' button.
colorTitle string The title to give to the color swatches.
shouldPadBottom boolean A flag that's true when padding should be alotted for the iOS tab bar, false if there should be no padding.
imageIsLoading boolean A flag that's true when the loading image should be displayed, false to display something else.
imageLoadWasSuccessful boolean Only matters if imageIsLoading is false. true to display the image of the product loaded from the server, false to show error text.
productName string The name of the product this view is presenting.
imageData string A base64 encoding of the image reprsenting the product.
rating number The number of stars (between 0 and 5) this product recieved.
availability string Indicates the availability of the product.
location string The department or asile the item is located in the store.
currentPrice string The current price of the product, whether the regular price or a discounted price. Displays in gold in the view.
oldPrice string The regular price for the product, usually when there is a discounted price present. If there is no discounted price, this property should be null. Displays in gray and crossed-out in the view.
colorData array An array of objects that represent the color options for a product. Use null for no colors.
optionsData object An object with a name property and a values property that represent options for the product (for example, shoe size, shirt size, etc.). Use null for no options.

Methods

(static) addToListClicked()

A function is called when the 'Add to List' button is clicked. This causes the route to switch momentarily to allow the native end to detect the change, and then switches the route back again so that the native can begin listening once more. Causes the button to be disabled briefly while the route resets.
