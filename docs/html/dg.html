Design Guidelines

Summit was created following the IBM Design Language V1 guidelines. You can see the PDF here.

Home

The home screen’s top section should show featured images or special sales. Below, the "Recommended" section should show recommended and popular products. Lastly, the “Shop All” section should show the user products or store departments.

Each department should be given a square image that fits in this section. If a department is featured it will appear in the featured images section at the top and not in "Shop All." Upon selection, a user should see products within that department.


Lists

After logging into the app, a user can create and manage multiple lists. A cumulative total should appear at the bottom showing how much each list will cost. On the right of each product users can see in which department they are located. Each store department will have its own beacon so the app can recognize a product’s location.

As a user moves throughout the store, their lists should update and reorder based on whichever department is closest to the user. After their list reorders, the user should be greeted with an in-app notification.


Log In

After logging in, a user should be able to see their list information. Without an account, a user will not be able to save any products to lists. However, they will have access to browse products in the app.


Product Detail Page

On this screen a user can learn more information about a product. Product images should either fit the width of the screen or have a white background. A user can see a product’s name, price, rating, details, color, and size options. If there is only one option for size the dropdown should not be displayed.


Notifications

Notifications should appear when a user has entered a store to welcome them and encourage phone use while shopping. Users should also receive notifications when a saved item becomes available or on sale.





Additional Screens

Here are some additional designs for the app, which could add value and functionality.

Search

A user could access “Search” at the top of every screen by selecting a search icon. A user could either filter by category or type in a desired product. The search will break down the store’s inventory into departments and categories.


Search Results

After searching for a product, a user could be greeted with this search results page. They could view this page as a two column grid, allowing for faster viewing, or in list view, for closer examination of products. A yellow dot could indicate whether the product is currently available at their store.

Users could also edit their search by selecting the refine button. This would pull up the search screen, complete with all categories and sub-categories.


My Store


On this screen, users could search for nearby store locations and set their favorite as their preferred store. By setting their preferred store, users will be able to see when products are available at that specific store. They can also see basic information like a store’s address, phone number, store-specific sales, or store events.

Cart

When a user adds a product to their cart they can either purchase it then or save it for later. If a store allows for phone purchases a user can scan their products and check out right from their phone, without having to wait in line. If a product is out of stock at their store, they can have it shipped to the store for pick up, or shipped to their home. They can also apply digital coupons that are synced with their account from this screen.


Product Detail Page Expansion

Besides a basic description found under product details, a user could also see reviews about a product and similar items. After a user has purchased a product, they can write a review and rate it in the review section.
